  <files>test/e2e/fresh-start.spec.ts, test/e2e/watchlist.spec.ts, test/e2e/trading.spec.ts, test/e2e/portfolio.spec.ts, test/e2e/chat.spec.ts, test/playwright.config.ts, test/docker-compose.test.yml</files>
  <action>
1. Ensure the Docker image "finally" is built (it should be from Plan 01). If not: `docker build -t finally .`

2. Start the test container using docker-compose:
   ```bash
   cd test && docker compose -f docker-compose.test.yml up -d --wait
   ```
   The `--wait` flag waits for the healthcheck to pass before returning. If it doesn't support `--wait`, use:
   ```bash
   docker compose -f docker-compose.test.yml up -d
   ```
   Then poll: `until curl -sf http://localhost:8000/api/health; do sleep 2; done`

3. Install Playwright test dependencies and browser:
   ```bash
   cd test && npm install && npx playwright install --with-deps chromium
   ```

4. Run the full Playwright test suite:
   ```bash
   cd test && npx playwright test
   ```

5. If any tests fail, diagnose using Playwright's error output and screenshots (saved on failure). Common failure causes:
   - **Selector mismatch**: UI element text or role doesn't match what the test expects. Fix the test selector OR the component (prefer fixing the test if the UI is correct).
   - **Timing**: Test doesn't wait long enough for async data. Increase timeout or add better wait conditions (prefer `expect().toBeVisible({ timeout })` over `waitForTimeout`).
   - **Shared state**: Earlier test's side effects affect later tests. Tests run sequentially (fullyParallel: false) and share a single database. Ensure test order handles cumulative state.
   - **Mock LLM response mismatch**: Chat tests expect specific patterns like "trading assistant" or "portfolio is worth". Verify the mock responses in `backend/app/llm/mock.py` match test expectations.
   - **Container not ready**: If first test fails but others pass, the healthcheck may pass before the app is fully initialized. Add a longer initial wait.

6. Fix failures iteratively. After fixes, re-run: `npx playwright test`

7. Once all 14 tests pass, tear down the container:
   ```bash
   cd test && docker compose -f docker-compose.test.yml down
   ```

8. Run the tests one final time from scratch (rebuild container) to confirm reproducibility:
   ```bash
   cd test && docker compose -f docker-compose.test.yml up -d --wait && npx playwright test && docker compose -f docker-compose.test.yml down
   ```

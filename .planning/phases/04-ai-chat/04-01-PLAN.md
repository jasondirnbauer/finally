From frontend/lib/api.ts (existing pattern to follow):
```typescript
async function apiFetch<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE}${path}`, options);
  if (!res.ok) {
    const text = await res.text().catch(() => res.statusText);
    throw new Error(`API ${path} failed ${res.status}: ${text}`);
  }
  return res.json() as Promise<T>;
}
```

From backend/app/routes/chat.py (API contract):
- POST /api/chat: body `{message: string}`, returns `{message: string, trades: [...], watchlist_changes: [...]}`
- GET /api/chat/history: returns `{messages: [{id, role, content, actions (JSON string|null), created_at}]}`

Backend response shapes for trade results (from _execute_actions in service.py):
- Success: `{ticker, side, quantity, price, status: "executed"}`
- Failure: `{ticker, side, error: "reason"}`
- Watchlist success: `{ticker, action, status: "done"}`
- Watchlist failure: `{ticker, action, error: "reason"}`
